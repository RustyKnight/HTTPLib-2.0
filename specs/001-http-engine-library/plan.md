# Implementation Plan: HTTPClient Library

**Branch**: `001-http-engine-library` | **Date**: 2026-06-28 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-http-engine-library/spec.md`
**Plan Notes**: `Features/Plan-HTTPLibrary.md`

## Summary

Build `HTTPClient`, a reusable Swift 6 HTTP library exposing GET, POST, PUT, and DELETE via
`URLSession` / `URLRequest` on macOS 14+. The library follows a progressive-disclosure API
(URL-only minimum; all other parameters optional) and surfaces all failures as typed throws.
POST additionally supports multipart form-data upload via a dedicated `[FormItem]` overload.
All operations are `async`/`await`, respect task cancellation, and return an `HTTPResponse`
carrying the HTTP status code and an optional raw response body.

**Technical approach**: A thin wrapper around `URLSession.data(for:delegate:)`. Public types
are value types (structs / enums) where possible; all types that cross concurrency boundaries
conform to `Sendable`. No third-party dependencies. `URLProtocol` subclassing provides test
isolation without requiring a protocol abstraction over `URLSession` in the public API.

## Technical Context

**Language/Version**: Swift 6.0 (`swift-tools-version: 6.0` in `Package.swift`)

**Primary Dependencies**: Foundation (`URLSession`, `URLRequest`, `HTTPURLResponse`, `Data`) ‚Äî no third-party deps

**Storage**: N/A

**Testing**: Swift Testing (`import Testing`, `@Test`, `@Suite`) ‚Äî preferred per `Features/Plan-HTTPLibrary.md`; existing test scaffold already uses Swift Testing syntax. See Constitution Check ¬ßII and Complexity Tracking.

**Target Platform**: macOS 14+ (as declared in `Package.swift`)

**Project Type**: Swift Package ‚Äî reusable library distributed exclusively via SPM

**Performance Goals**: Delegate entirely to `URLSession`; library introduces no artificial latency, buffering, or thread hops beyond what Foundation provides.

**Constraints**: Zero compiler warnings at Swift 6.0; no force-unwraps in public API (`!` prohibited); all public types that cross concurrency boundaries must conform to `Sendable`; no third-party networking dependencies.

**Scale/Scope**: Small, focused library ‚Äî ~6 source files, ~9 test files, zero external services required in tests.

**Structure Decision**: Standard Swift Package Manager layout. `Sources/HTTPLib/` contains all
public API files (one type per file) plus an `Internal/` sub-directory for implementation helpers
that are not part of the public API surface. `Tests/HTTPLibTests/` mirrors concerns with one
test file per public surface area, plus `Helpers/` for shared test infrastructure
(`MockURLProtocol`). `HTTPLib.swift` (the SPM-generated stub) is superseded by the files above;
its `HTTPLib` struct placeholder should be removed when `HTTPClient` is introduced.
